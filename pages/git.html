Como funciona?

Como dito anteriormente, o Git se diferenciava dos demais. Na época em que foi lançado, a maioria dos sistemas enxergava as mudanças como uma lista que deveria ser armazenada, sendo assim, toda vez que uma mudança e o salvamento ocorria, algo novo seria adicionado na lista. O Git oferece algo bem diferente, trabalhando com uma ideia parecida com um snapshot (print), onde todo o arquivo será salvo em algo que seria como uma foto tirada e se não houver mudanças, nada será salvo.

Outra ideia trabalhada pelo Git são as Branches, que nada mais são do que ramificações feitas durante o desenvolvimento de algum software. A ideia de uma Branch é auxiliar o desenvolvedor durante o momento em que ele precisa criar outras versões do mesmo código sem alterar o código principal ou quando vários desenvolvedores trabalham no mesmo projeto. Diversos sistemas de controle de versão possuem apoio para branch, mas o Git se destaca pelo sua leveza e facilidade, estimulando o usuário a trabalhar com isso.

Definições básicas

  • Repositório: Saber o que é um repositório é muito importante para entender o Git, esse termo é utilizado para o arquivo principal, é o local que armazena todas as partes do código que foram commitadas.

  • Três estados do Git:
    Existem três estados em que o código pode ficar em relação ao Git:
    • Modificado: Nesse estado o código recebeu mudanças, mas não foi mandado para o salvamento
    • Preparado: Nesse estado as mudanças feitas, ficam preparas e esperando o commit
    • Commitado: Nesse estado as mudanças e o arquivo foi commitado e já está salvo

  • Git-Init: O init é o comando que pede que o Git crie um repositório novo.
  • Git-Add: Add manda suas mudanças para o estado de preparado, onde esperara para ser commitado.
  • Git-Commit: O principal comando do git, que tira a foto do arquivo e salva.
  • Git-Merge: Esse comando junta todas as mudanças feitas em branches diferentes.
  • Git-Push: Ao utilizar um repositório online, o push manda repositório local (do computador) para o servidor online.
  • Git-Pull: Ao utilizar um repositório online, o pull pega o repositório do servidor online e manda para o repositório local.
