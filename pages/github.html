Como funciona?

Ao fazer o cadastro na plataforma o usuário tem a opção de criar um repositório novo e fazer o commit do código que está em desenvolvimento no repositório local para o repositório novo criado no Github. O Github vai armazenar o repositório e deixar que o usuário escolha deixar visível para outros ou não, ganhando destaque por sua rapidez, que faz com que os usuários consigam acompanhar o código em tempo

Outro ponto importante é a ideia de rede social que o Github possui, possibilitando aos usuários compartilhar repositórios e até mesmo usar o repositório de outra pessoa para si.

Definições básicas

  • Main e Branch: Ao se trabalhar com variadas versões do mesmo código é possível criar ramificações denominadas branches, com main sendo a principal e final versão.

  • Git-Remote: Faz a conexão entre o Git e o Github
  • Git-Commit: O principal comando do git, que tira a foto do arquivo e salva.
  • Git-Merge: Esse comando junta todas as mudanças feitas em branches diferentes.
  • Git-Push: Ao utilizar o Github, o push manda o repositório local (do computador) para o servidor online.
  • Git-Pull: Ao utilizar o Github, o pull pega o repositório do servidor online e manda para o repositório local.
